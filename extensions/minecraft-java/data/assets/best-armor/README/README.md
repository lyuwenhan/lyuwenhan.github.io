# Best Armor

A lightweight **client-side Fabric mod** that adds three prebuilt administrator shulker boxes and a Void Sword to the operator creative tab.

## Overview

Best Armor adds four generated items to the operator-only creative tab:

- Yellow Shulker Box
- Orange Shulker Box
- Red Shulker Box
- Void Sword

The three shulker boxes contain different equipment presets:

- The yellow shulker box contains high-end equipment based primarily on enchantment levels obtainable through normal Survival gameplay.
- The orange shulker box keeps a mostly vanilla-looking equipment profile while adding normally incompatible enchantments, extreme protection levels, unbreakable items, special boots, and a shield that blocks all registered damage types.
- The red shulker box contains full-power administrator equipment with extreme enchantments, custom combat attributes, increased attack range, fall-damage immunity boots, and other non-Survival item components.

## Key Features

- Adds three prebuilt equipment shulker boxes
- Adds a Void Sword
- Includes weapons, tools, armor, Elytra equipment, shields, and fishing rods

## Void Sword

The Void Sword is added directly to the operator creative tab as a separate item.

It is internally based on a netherite sword but uses the diamond sword item model.

### Display

- Name: `Void Sword`
- Name color: Light purple
- Name style: Bold
- Item model: Diamond Sword
- Enchantment glint: Always enabled
- Unbreakable

### Damage Behavior

The sword uses the `fell_out_of_world` damage type.

This is the same damage type associated with void damage rather than normal melee damage.

### Attributes

When held in either hand, the sword provides:

- Attack Damage: +2,147,483,647
- Attack Speed: +2,147,483,647

### Range

- Maximum configured attack range: 64 blocks
- Maximum configured creative attack range: 64 blocks

### Blocking

The sword also receives the same all-damage blocking component used by the administrator shield:

- Blocks every registered damage type
- Uses a 360-degree blocking angle
- Uses zero blocking delay
- Takes no durability damage while blocking
- Applies a very high damage-reduction value

Actual server behavior remains subject to server-side item validation and combat authority.

## Glideplate Compatibility

Each shulker box includes a specially configured netherite chestplate with Elytra functionality.

The chestplate receives:

- `glideplate_has_elytra` custom data
- `gliding` custom data
- `gliding_chestplate_has_elytra` custom data
- Glideplate-compatible custom model data
- The vanilla glider component
- The translated name `Netherite Chestplate with Elytra`
- The lore text `With Elytra`

These components allow the item to behave as a gliding chestplate without registering a new item type.

The generated chestplate uses compatibility data associated with [Glideplate](https://modrinth.com/mod/glideplate).

Installing Glideplate may improve related item naming, models, and visual presentation, but Best Armor generates the item stacks directly.

## Creative Tab Behavior

The generated items are added to the operator creative tab only when operator-restricted items are visible.

The following items are added to both the main operator-tab display and creative inventory search:

- Yellow Shulker Box
- Orange Shulker Box
- Red Shulker Box
- Void Sword

The items are generated only when a client world is available because the mod requires the active registry access to resolve enchantments and damage types.

## Technical Behavior

Best Armor directly creates `ItemStack` instances and assigns vanilla data components.

The mod uses components including:

- Enchantments
- Unbreakable
- Attribute Modifiers
- Container Contents
- Custom Data
- Custom Model Data
- Glider
- Item Name
- Item Lore
- Attack Range
- Blocks Attacks
- Damage Type
- Enchantment Glint Override
- Item Model

The mod does not:

- Register custom item types
- Register custom blocks
- Register commands
- Add configuration files
- Add screens
- Read item definitions from JSON or text files
- Execute `/give` commands
- Parse command strings

## Visual Behavior

- The yellow, orange, and red boxes appear as vanilla colored shulker boxes
- Each box uses a custom item name
- The three boxes appear in the operator creative tab and creative search
- The Void Sword uses the diamond sword model
- The Void Sword always displays an enchantment glint
- Glideplate-style chestplates include custom naming, lore, and model data
- No custom textures are included by this mod
- No custom user interface or HUD is added

## Multiplayer Safety

- Client-side only
- No server installation is required for the mod itself to load
- Server authority still controls inventory and combat behavior
- Servers may validate, alter, reject, or remove generated item stacks
- Unsupported enchantment combinations may not behave consistently on every server
- Custom attributes and data components may be rejected by server-side validation
- The Void Sword and administrator equipment can severely disrupt normal gameplay
- Use these items only in private worlds or on servers where administrator equipment is explicitly allowed
